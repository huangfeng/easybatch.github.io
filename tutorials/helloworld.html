Hello World Tutorial

1. Introduction

This tutorial will show you how to build a very simple batch application to process a in-memory data source. We will process text records containing tweets and simply print them out to the console (remember, it is a hello world sample ). Here are the tweets:

1,foo,easy batch rocks! #EasyBatch
2,bar,@foo I do confirm :-)

Each record is composed of:

  • The tweet id
  • The user name
  • The tweet content

Let's get started!

2. What youâ€™ll need

  • JDK 1.6+
  • Maven
  • Git (Optional)
  • Your favorite IDE (Optional)

3. Reading tweets

You need to read records from a String, you can use the built-in StringRecordReader. This reader will read the data source line by line and produce StringRecord instances which, as you may have guessed, contain strings as payload, or tweets in our case. Let's configure an engine to use this reader:

String dataSource = "1,foo,easy batch rocks! #EasyBatch\n" +
                    "2,bar,@foo I do confirm :-)";

Engine engine = new EngineBuilder()
    .reader(new StringRecordReader(dataSource))
    .build();

4. Processing tweets

With Easy Batch, you only write your application's logic and let the framework take care of the boilerplate code of reading, parsing, logging, reporting, etc.

Let's implement the business logic with a RecordProcessor to process StringRecords:

TweetProcessor.java
public class TweetProcessor implements RecordProcessor<StringRecord, StringRecord> {

    @Override
    public StringRecord processRecord(StringRecord record) throws Exception {
        System.out.println(record.getPayload());
        return record;
    }
}

This processor will simply print out tweets to the console as required.

Heads up!

The record processor returns the record at the end of processRecord() method. This allows the record to be piped out to the next processor in the pipeline if any. We have only one processor in this tutorial, you will see how to build a data processing pipeline later on in this tutorial.

5. Putting it all together

Now that you have implemented the application's business logic, you should configure Easy Batch engine to:

  1. Read records from the String data source
  2. Process each record using the TweetProcessor

The following listing provides this configuration:

public class Launcher {

    public static void main(String[] args) throws Exception {

        // Create the String data source
        String dataSource =
            "1,foo,easy batch rocks! #EasyBatch\n" +
            "2,bar,@foo I do confirm :-)";

        // Build a batch engine
        Engine engine = new EngineBuilder()
            .reader(new StringRecordReader(dataSource))
            .processor(new TweetProcessor())
            .build();

        // Run the batch engine
        Report report = engine.call();

        // Print the batch execution report
        System.out.println(report);
    }
}

To run the engine, you call the call method.

Easy Batch returns a report at the end of execution containing various metrics and statistics. In this example, its content is printed out to the console to examine it. Time to run the tutorial and see the result.

6. Running the tutorial

6.1 From the command line

To run the tutorial from scratch, you can follow these instructions:

$>git clone https://github.com/EasyBatch/easybatch-tutorials.git
$>cd easybatch-tutorials
$>mvn install
$>mvn exec:java -PrunHelloWorldTutorial

You should see something like the following output in the console:

INFO: Initializing easy batch engine
INFO: Data source: In-Memory String
INFO: Strict mode: false
INFO: Total records = 2
INFO: easy batch engine is running...
1,foo,easy batch rocks! #EasyBatch
2,bar,@foo I do confirm :-)
INFO: Shutting down easy batch engine

6.2 From your IDE

First, you need to checkout the source code of the tutorial available here:

$>git clone https://github.com/EasyBatch/easybatch-tutorials.git

If you do not have git installed, you can download a zip file containing the project's source code from GitHub here.

Then, import the easybatch-tutorials module in you favorite IDE and resolve maven dependencies.

Finally, run the org.easybatch.tutorials.basic.helloworld.Launcher class without any argument.

7. Summary

Congratulations! You have built your first batch application with Easy Batch. You learned how to create, configure and run a batch engine to process data from a data source.

All you have done is:

  • Implementing the business logic of your application
  • Providing configuration meta data

Easy Batch handled the boilerplate code of reading, parsing, logging and reporting for you.

Key APIs Tutorial

1. Introduction

After reading the Hello world tutorial, you should be familiar with basic blocs to create an application with Easy Batch. This tutorial will guide you step by step through the key APIs to show you how to use them efficiently. We will continue our journey with processing the same tweets but this time we will read them from a CSV flat file:

tweets.csv
id,user,message
1,foo,easy batch rocks! #EasyBatch
2,bar,@foo I do confirm :-)
This tutorial is divided into 5 parts. Each part will introduce a new API and give an example of how to use it.

2. What youâ€™ll need

  • JDK 1.6+
  • Maven
  • Git (Optional)
  • Your favorite IDE (Optional)

3. Reading records

Unlike the hello world tutorial where tweets were read from a in-memory string, in this tutorial we will read data from a CSV flat file. Easy Batch provides the FlatFileRecordReader to read records from a flat file. So let's use this reader:

Engine engine = new EngineBuilder()
    .reader(new FlatFileRecordReader(new File("tweets.csv")))
    .build();

4. Filtering records

The RecordFilter allows you to filter records from being processed. In this tutorial, you would like to filter the header record containing field names.

Easy Batch comes with several built-in filters for common use cases and you can implement your own filter as well. To filter the header record, you can use the built-in HeaderRecordFilter designed specifically for that:

Engine engine = new EngineBuilder()
    .reader(new FlatFileRecordReader(new File("tweets.csv")))
    .filter(new HeaderRecordFilter())
    .build();

This snippet will tell the engine to skip the first record.

5. Mapping records

Easy Batch development is POJO-centric, you would like to map each tweet to an instance of the Tweet bean.

Each field in the text record should be mapped to the corresponding field in the Tweet bean.

You can provide object mapping logic by implementing the RecordMapper interface.

Easy Batch comes with a built-in DelimitedRecordMapper which is able to map delimited values to a java bean. It sounds like this is exactly what you want since you will be reading data from a CSV file.

Tweet.java
public class Tweet {

    private int id;

    private String user;

    private String message;

    // Getters, Setters and toString omitted
}

Let's register the DelimitedRecordMapper:

Engine engine = new EngineBuilder()
    .reader(new FlatFileRecordReader(new File("tweets.csv")))
    .filter(new HeaderRecordFilter())
    .mapper(new DelimitedRecordMapper(Tweet.class, new String[]{"id", "name", "message"}))
    .build();

You must specify the type of the target bean and an array of field names in the same order as in the CSV file so that the mapper can bind data respectively to each field. By default, this mapper uses the comma "," as delimiter which is fine in our case. You can also configure a custom delimiter if you want.

Note that the mapper also converts raw textual data to typed data in the Tweet instance.

Heads up!

If the mapper is not able to map a record to an instance of the Tweet class, it will throw an exception that will cause the engine to ignore this record and move to the next one.

6. Validating records

We all know that tweets can have at most 140 characters, and we would like to process only valid tweets.

How to declare this constraint on our Tweet bean? The Bean Validation API is the way to go since it provides an elegant solution to declare validation constraints on domain objects using annotations.

Easy Batch provides support for Bean Validation API so you can just annotate your Tweet class with constraint annotations.

Now that you placed validation constraints on the Tweet bean, let's configure the engine to validate records according to these constraints:

Tweet.java
public class Tweet {

    private int id;

    @NotNull
    private String user;

    @Size(min=0, max=140)
    private String message;

    // Getters, Setters and toString omitted

}
Engine engine = new EngineBuilder()
    .reader(new FlatFileRecordReader(new File("tweets.csv")))
    .filter(new HeaderRecordFilter())
    .mapper(new DelimitedRecordMapper(Tweet.class, new String[]{"id", "name", "message"}))
    .validator(new BeanValidationRecordValidator<Tweet>())
    .build();

The BeanValidationRecordValidator is provided by Easy Batch and is responsible of validating records. When a record is not valid, the engine will reject it and move to the next one.

7. Processing tweets, finally!

At this stage, you have filtered undesired data, mapped tweets to domain objects and validated them. You should now be able to process tweets quietly.

You have seen in the hello world tutorial a simple RecordProcessor implementation that prints out tweets to the console. In this tutorial, you will learn how to use the ComputationalRecordProcessor API, that is, a record processor that performs a computation.

You would like to count the number of tweets containing the hash tag #EasyBatch, this is the business logic of your application. It is time write some code :

TweetCountProcessor.java
public class TweetCountProcessor implements ComputationalRecordProcessor<Tweet, Tweet, Integer> {

    private int count;

    @Override
    public Tweet processRecord(Tweet tweet) {
        if (tweet.getMessage().contains("#EasyBatch")) {
            count++;
        }
        return tweet;
    }

    @Override
    public Integer getComputationResult() {
        return count;
    }
}

This processor is straightforward and will do the trick. The result of computation can be obtained at the end of the batch from the execution report. You will see how to do that in a second, for now let's register this processor within the engine:

Engine engine = new EngineBuilder()
    .reader(new FlatFileRecordReader(new File("tweets.csv")))
    .filter(new HeaderRecordFilter())
    .mapper(new DelimitedRecordMapper(Tweet.class, new String[]{"id", "name", "message"}))
    .validator(new BeanValidationRecordValidator<Tweet>())
    .processor(new TweetCountProcessor())
    .build();

8. Putting it all together

To launch the tutorial, you will use the following class:

public class Launcher {

    public static void main(String[] args) throws Exception {

        // Input file tweets.csv
        File tweets = new File(args[0]);

        // Build a batch engine
        Engine engine = new EngineBuilder()
            .reader(new FlatFileRecordReader(tweets))
            .filter(new HeaderRecordFilter())
            .mapper(new DelimitedRecordMapper(Tweet.class, new String[]{"id", "name", "message"}))
            .validator(new BeanValidationRecordValidator<Tweet>())
            .processor(new TweetCountProcessor())
            .build();

        // Run the batch engine
        Report report = engine.call();

        // Print the batch execution report
        System.out.println("Total tweets containing #EasyBatch: " + report.getBatchResult());
    }
}

8.1 Run the tutorial from the command line

To run the tutorial, proceed as follow:

$>git clone https://github.com/EasyBatch/easybatch-tutorials.git
$>cd easybatch-tutorials
$>mvn install
$>mvn exec:java -PrunKeyAPIsTutorial

You should see the following output:

INFO: Initializing the engine
INFO: Data source: tweets.csv
INFO: Strict mode: false
INFO: The engine is running
INFO: Record #1
  [Record{number=1, payload='id,user,message'}]
  has been filtered.
Total tweets containing #EasyBatch: 1
INFO: Shutting down the engine

8.2 Run the tutorial from your IDE

First, you need to checkout the source code of the tutorial available here:

$>git clone https://github.com/EasyBatch/easybatch-tutorials.git

If you do not have git installed, you can download a zip file containing the project's source code from GitHub here.

Then, import the easybatch-tutorials module in you favorite IDE and resolve maven dependencies.

Finally, run the org.easybatch.tutorials.basic.keyapis.Launcher class without any argument.

9. Summary

In this tutorial, you have learned how to use most of the common APIs of Easy Batch. You have seen how to filter, map and validate tweets before processing them.

All you have done is:

  • Implementing the business logic of your application
  • Providing data validation constraints
  • Providing configuration meta data

Easy Batch handled all the boilerplate code of reading, filtering, parsing, mapping and validating records.

The following table summarizes what you have done and what Easy Batch has done for you:

Task You Easy Batch
Provide configuration meta data
Provide validation constraints
Provide business processing logic
Handle input file I/O
Read input file record by record
Filter undesired records
Map records to domain objects
Validate records
Apply business processing logic
Log errors
Report metrics and statistics

Until now, you have always registered only one record processor. In the next tutorial, you will learn how to build a unix-like data processing pipeline with several processors.

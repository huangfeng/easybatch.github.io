Frequently Asked Questions

1. How does Easy Batch read data from a data source?

Easy Batch streams data record by record from the data source. Depending on the data source type, a record can be a line in a flat file, an tag in an Xml file, a record in a database table, etc.

The RecordReader abstraction is intended to be implemented with a Streaming API so that the data source is not entirely loaded in memory (which is the main cause of java.lang.OutOfMemoryError of many batch applications).

Easy Batch comes with several built-in implementations of the RecordReader interface to read data from a variety of data sources. Please refer to the user guide for all details about available record readers.

2. Can I use Easy Batch to process non-textual data?

Yes. Even though common use cases are about processing textual data, Easy Batch Record abstraction can be implemented for any type of input data. For example, in a scenario where you need to compress a set of images with Java in a batch mode, a record can be one image file.

Easy Batch API is generic and can be used to process any type of input data.

3. Can I use the Bean Validation API (JSR 303/349) with Easy Batch?

Yes. Easy Batch uses the reference implementation Hibernate validator to validate domain objects.

For all details about how to validate data using Bean Validation API with Easy Batch, please refer to the user guide (see "Validating data" section). You can also find a complete tutorial of this feature here.

4. Does Easy Batch implement the Batch Processing for Java Platform (JSR 352)?

No. Easy Batch has been designed and implemented before the JSR 352 has been submitted.

5. Can I monitor Easy Batch execution with JMX?

Yes. By default, Easy Batch enables a JMX MBean named org.easybatch.core.jmx:type=MonitorMBean at startup. You can use any standard JMX compliant tool to monitor Easy Batch metrics. For more details about this feature, please refer to the JMX tutorial.

6. Can I use Easy Batch with Spring ?

Yes. Easy Batch provides a Spring factory bean that allows you to configure and declare Easy Batch engine as a Spring bean. You can find a complete example of using Easy Batch with Spring here.

7. How does Easy Batch compare to Spring Batch?

Spring Batch is an advanced batch processing framework with a steep learning curve. Entire books have been written on Spring Batch because of its very rich and advanced features set such as remoting, partitioning, automatic retry on failure, etc. Easy Batch is a simple and lightweight framework that can be learned quickly an used easily for the majority of batch processing use cases.

The main difference between Easy Batch and Spring batch is in the processing workflow: Spring Batch is "Chunk Oriented" (a set of Read-Process-Write steps on a chunk of input data) while Easy Batch is "One Shot" (the data source is processed entirely record by record, the unix way, with a predefined and extensible workflow).

The second difference is that Easy Batch does not use concepts such as Job, JobExecution, JobInstance, Step, StepExecution used by Spring Batch and the JSR 352 in general. Easy Batch is not intended to compete with Spring Batch but to provide a lightweight alternative that is easy to learn and use.

8. How can I configure Easy Batch logging ?

Easy Batch framework core uses the java.util.logging API. This choice has been made because one of the goals of Easy Batch is to have no dependencies. By default, Easy Batch logs messages to the standard output.

If you use Java SE 6 and prior versions, you should implement the java.util.logging.Formatter interface and register your custom formatter at your application's startup. Since Java SE 7, you can use the property java.util.logging.SimpleFormatter.format as a JVM parameter (or define it in logging.properties file) to specify the log message format. Please refer to official documentation of the the java.util.logging API for more details about how to customize it.

9. Is there a micro benchmark for Easy Batch?

Yes, here it is. It uses JMH to measure the potential overhead of Easy Batch.

You can follow detailed instructions to launch the benchmark and see results.

Release notes

Current version : 3.0.0 (2015-03-01)

This is the best release of Easy Batch framework so far. This major release has added/removed/renamed many classes and interfaces to provide a more flexible and consistent API.

The main features of this new release are:

  • Introduction of the processing pipeline in the framework's core. This is the most important feature of this version
  • Introduced the filter chain in the framework's core
  • Introduction of several extension points:
    • Add pre/post processing before and after each step of the workflow (Pull request #35)
    • Add custom ignored/rejected/error record handlers (Pull Request #34)
  • Introduction of the AbstractRecordDispatcher API for more efficient parallel processing
  • Adding support for asynchronous processing through the new JMS module
  • Adding support for reading data using JPA
  • Adding support for reading data from MongoDB
  • Adding support for streaming JSON data sources through the Json-P API reference implementation
  • Adding integration modules with OpenCsv and Apache commons CSV for flat file parsing
  • Adding integration module with Xstream for Object-XML data binding
  • Adding integration modules with Jackson and Gson for Object-JSON data binding
  • Adding integration module with Spring JDBC for ResultSet-Object data binding

This release has also fixed many bugs and significantly improved tutorials and documentation.

Please check the change log for all details about changes.

Prior versions

Version : 2.2.0 (2014-06-08)

This release comes after important changes in Easy Batch online publication. This release has:

  • Changed website URL to http://www.easybatch.org
  • Changed base package form io.github.benas.easybatch to org.easybatch for all APIs
  • Changed maven group id to org.easybatch
  • Moved benchmark classes to a dedicated module easybatch-bench which now uses JMH for microbenchmarks
  • Fixed issue #29 : EasyBatchReport problem while processing large volume files
  • Fixed issue #30 : fix java.io.FileNotFoundException when running the Spring tutorial
  • Added tutorial about processing JSON data source

Version 2.1.3 (2014-02-24)

This release is a third bug fix release of the 2.x branch. This release has:

  • Fixed issue #27: ObjectMapper throws a NPE when using JDBC column aliases
  • Fixed issue #28: ObjectMapper throws a NPE when there is no Pojo field corresponding to a given database column
  • Added the JMS tutorial
  • Added the EasyBatchReportsAggregator API to merge partial reports
  • Added the strict mode parameter to abort execution on first error
  • Introduced some minor API changes

Version 2.1.2 (2014-02-16)

This release is a second bug fix release of the 2.x branch. This release has:

  • Added a tutorial with an advanced ETL application
  • Added a tutorial with a custom record reader of a master-detail delimited flat file
  • Fixed some minor bugs

Version 2.1.1 (2014-02-01)

This release is mainly a bug fix release with 10+ fixed bugs. It also includes some minor API changes and documentation updates:

  • Renamed DsvRecordMapper to DelimitedRecordMapper
  • Added a convention over configuration for DelimitedRecordMapper to read field names from the header record
  • Added a convention over configuration for JdbcRecordMapper to read field names from the result set meta data
  • Added a Spring factory bean to use Easy Batch engine as a Spring bean
  • Added a performance benchmark
  • Improved RecordReader API design
  • Improved Object Mapper implementation
  • Improved code quality by fixing 50+ sonar warnings

Please check the change log for all details about changes and fixed bugs.

Version 2.1.0 (2014-01-20)

This release adds the following new features:

  • Added a new module to process data from a database using JDBC API
  • Added a new module to process data from an xml file using StAX and JAXB APIs
  • Added a new API to read data from an in-memory String data source
  • Added a new API to read data from the Standard Input
  • Added data source name to Easy Batch report
  • Added 10 new tutorials about all Easy Batch features (basic, intermediate and advanced)
  • Updated documentation and Javadoc

Please check the change log for all details about changes.

Version 2.0.0 (2014-01-12)

This is the first stable version of Easy Batch in the 2.x branch. This version brings a big refactoring to the former CB4J framework to make it able to process not only CSV input but also any type of input. This release adds many new features such as:

  • New Record interface for generic input data type
  • Record filtering through the RecordFilter API which allows to skip records from being processed by the engine
  • Redesigned validation approach which is now well integrated with the Bean Validation API (JSR303)
  • Simplified logging through a single and centralised log file
  • New EasyBatchReportFormatter API to format Easy Batch reports to a specific format (HTML, PDF, etc)
  • Updated documentation and Javadoc

Please check the change log for all details about changes.

Version 1.4.0 (2013-09-21)

This release adds many new features such as:

  • New BatchResultHolder API to be able to get computation result at the end of batch execution
  • New CSV/Object mapper implementation to map records to domain objects based on field names declared in header record
  • New parameter to abort execution on first record processing error / record mapping exception
  • New parameter to enable/disable JMX monitoring feature
  • New validators to validate that date value is before/after a given date
  • New validators to validate that field field content size has a min/max length

Please check the change log for all details about changes.

Version 1.3.0 (2013-04-21)

This release adds the following new features:

  • Basic record processing transaction support
  • New JMX custom clients : command line and graphical asynchronous real time monitors
  • New batch configuration builder API

Please check the change log for all details about changes.

Version 1.2.0 (2013-01-13)

This release adds the following new features:

  • Fixed-Length Record parsing support
  • Quartz scheduling module
  • Hibernate integration use case
  • Spring integration use case

This release also includes some minor API changes and documentation updates. Please check the change log for all details about changes.

Version 1.1.0 (2012-12-08)

This release adds the following new features:

  • Support for data qualifier character
  • Support for whitespaces trimming
  • Support for execution abort on first reject
  • JMX monitoring support
  • More precise reporting about records processed with errors

For all details about changes in this release, please refer to the change log.

Version 1.0.2 (2012-12-01)

This is the second maintenance release in the 1.0.x branch. This release adds some minor changes to support loading configuration parameters file from the classpath. It also fixes a performance issue due to String concatenation. For all details about changes in this release, please refer to the change log.

Version 1.0.1 (2012-11-01)

This is the first maintenance release in the 1.0.x branch. The release includes multiple performance optimizations, bugfixes and documentation updates. The full list of changes may be found in the change log.

Version 1.0.0 (2012-10-14)

This is the first CB4J release. The main features of this release are :

  • Initial core API and Implementation
  • Common data type validators
